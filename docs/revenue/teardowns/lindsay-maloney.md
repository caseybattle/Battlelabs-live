# Async Teardown: Lindsay Maloney

## What is working
- The private coaching application is clear and aligned with a premium offer.
- The page already signals enough authority that a fit buyer understands the offer quickly.
- The application path likely works for high-intent visitors.

## Likely conversion leak
- The application step is still high-friction for creator-operators who are interested but not fully ready.
- That usually creates a gap between curiosity and commitment where warm leads disappear.
- The page would likely benefit from a smaller async first move that captures intent before the full application.

## One change to make first
Add a softer async entry before the full application:
- one smaller fit-check form
- one “not ready to apply yet” capture path
- one follow-up sequence that moves warm leads toward the full application

## What Battle Labs would build
- lighter async CTA before the full application
- short readiness form
- follow-up sequence for warm but not-ready leads
- cleaner routing into the existing private coaching application

## Tight response version
The strongest opportunity is not rewriting the offer, it is rescuing warm interest before the full application becomes the only next step. I would start with a softer async entry path and follow-up sequence that nudges stronger-fit leads into the existing application. If you want, I can scope that into a pilot build and keep the whole process async.
