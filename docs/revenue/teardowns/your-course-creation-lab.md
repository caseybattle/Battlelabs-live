# Async Teardown: Your Course Creation Lab

## What is working
- The business clearly speaks to course creators and education-focused buyers.
- The offer direction is understandable, which means the main issue is likely qualification and follow-up rather than messaging from scratch.
- The page likely attracts buyers at different stages, which is useful if the intake path can sort them well.

## Likely conversion leak
- Different creator stages and support needs may be flowing into one path too early.
- That can make it harder to pre-qualify people before live contact.
- Warmer leads who are interested but not ready to book direct help may not have a smaller async entry step.

## One change to make first
Create an async intake layer that sorts by course stage and need:
- planning
- building
- launching
- optimizing

Then connect each route to a follow-up path that fits the creator’s readiness.

## What Battle Labs would build
- segmented inquiry path by course stage
- cleaner CTA framing around readiness
- short follow-up sequence for each segment
- simpler review workflow before any live conversation

## Tight response version
I would not start by rebuilding the whole offer. I would start by sorting creator-stage intent earlier, then tying follow-up to that segment so interested people do not all hit the same path too soon. If you want, I can scope that into a pilot build and keep the whole process async.
