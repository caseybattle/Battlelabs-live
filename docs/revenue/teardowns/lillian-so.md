# Async Teardown: Lillian So

## What is working
- The application page is persuasive and clearly tied to a premium offer.
- The positioning is strong enough that the main problem is not trust.
- High-intent visitors can still see what the next step is.

## Likely conversion leak
- The page leans heavily on a high-friction application before there is a softer async next step.
- That usually creates a drop-off point for interested buyers who are not ready to commit at that level immediately.
- Without a lighter entry point, warm prospects may disappear before follow-up has a chance to work.

## One change to make first
Add a softer async qualification step before the full application:
- one short pre-application form
- one smaller “is this right for me?” path
- one follow-up sequence for interested but not-ready applicants

## What Battle Labs would build
- lighter pre-application CTA
- short async form for fit and readiness
- follow-up sequence for warm leads who do not complete the full application
- simple routing so stronger-fit applicants are easier to review

## Tight response version
The offer is already persuasive, so I would not start by changing the whole page. I would start by adding a softer async step before the full application so warm leads do not fall out before they are ready. If you want, I can scope that into a pilot build and keep the whole process async.
