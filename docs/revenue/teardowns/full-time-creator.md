# Async Teardown: Full Time Creator

## What is working
- The page has strong social proof and a clear offer.
- The brand already carries enough authority that the main challenge is likely conversion flow, not credibility.
- The audience is likely motivated, which makes even small front-end improvements valuable.

## Likely conversion leak
- The join process appears to ask for a relatively heavy commitment before a lighter async qualification step is visible.
- That can create friction for interested creators who want to understand fit before moving deeper into the process.
- A heavier form-first path also tends to reduce the amount of useful follow-up data captured early.

## One change to make first
Insert a softer async qualification step before the heaviest join motion:
- one short “is this right for you?” intake
- one split based on creator stage or current business model
- one follow-up sequence that nudges people toward the main join path

## What Battle Labs would build
- lighter pre-qualification path before the main form
- creator-stage routing for better segmentation
- follow-up sequence for warm but not ready leads
- clearer handoff from interest to application or join step

## Tight response version
The offer already has strong proof, so the more interesting gap is likely the amount of friction before someone can raise their hand. I would start by adding a softer async qualification path ahead of the heavier join flow, plus follow-up tied to creator stage. If you want, I can scope that into a pilot build and keep the whole process async.
