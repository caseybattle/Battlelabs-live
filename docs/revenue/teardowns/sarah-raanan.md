# Async Teardown: Sarah Raanan

## What is working
- The page communicates clear authority and two distinct service paths.
- The reader can tell there is a real business coaching offer behind the page.
- The call-to-action is not hidden, so high-intent visitors can still act.

## Likely conversion leak
- Both offers appear to push people toward live booking before they have an easier async way to sort themselves.
- Business coaching and content strategy likely attract different buyer intent, but there is not enough visible segmentation before the call decision.
- Visitors who are interested but not ready to book live may leave without entering any follow-up flow.

## One change to make first
Add an async qualification layer before the call:
- one lightweight form split by offer type
- one short “best fit” question set
- one automated follow-up path for each service

That gives Sarah a better way to capture warmer traffic that is curious but not ready to commit to a call immediately.

## What Battle Labs would build
- segmented inquiry form for coaching vs content strategy
- shorter page-level copy tied to each path
- follow-up email logic based on the selected offer
- simple handoff system so qualified leads are easier to review asynchronously

## Tight response version
What is working is the authority and the fact that both offers are visible. The likely leak is that both paths still rely too heavily on live booking before the visitor gets an easier async next step. I would start by adding a segmented inquiry layer and follow-up path for each service. If you want, I can scope that into a pilot build and keep the whole process async.
