# Close and Kickoff Playbook

## Objective
Move a warm prospect from:
- accepted pilot scope
- to payment
- to kickoff

without adding calls or a long custom back-and-forth.

## Close sequence
1. Send the pilot proposal
2. If they say yes, send the pre-payment message
3. Send the payment link
4. Once paid, send the kickoff message
5. Move the lead from `proposal sent` to `won`

## Minimal rule
Do not start work before:
- scope is accepted
- payment path is confirmed
- required access or assets are requested

## What must be collected at kickoff
- page URLs
- current form or tool links
- copy/assets needed for the pilot
- access requirements if any
- one sentence defining the exact bottleneck being fixed

## Scope guardrail
If the client starts expanding scope at close, respond with:

> I want to keep the pilot tight so it ships quickly and proves value. Once that first bottleneck is fixed, we can scope the next layer from there.

## After payment
Immediately send:
- kickoff message
- access checklist
- expected timeline
- reminder of what is intentionally not included
