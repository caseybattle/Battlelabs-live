# Payment Setup Checklist

## Objective
Make Battle Labs sellable without adding calls or manual invoicing as the only path.

## Preferred order
1. Stripe payment link for pilot builds
2. simple proposal acceptance flow
3. optional Gumroad or product checkout for microproducts

## Minimum Battle Labs payment setup
- one pilot-build payment link
- one higher-tier custom-build placeholder link or invoice path
- one confirmation message or email after payment
- one internal step that moves the lead from `proposal sent` to `won`

## Suggested pilot link naming
- `Battle Labs Pilot Build`
- `Battle Labs Lead Capture Pilot`

## Pricing checkpoints
- start with a pilot in the `$250-$500` range
- raise after 2 closes if the work remains narrow and fast
- do not create too many checkout options before the first win

## Blocker
This is the main remaining revenue blocker that still needs user-controlled account access.
