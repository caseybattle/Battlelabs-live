# Closing Checklist

## Before sending payment
- proposal matches the teardown
- price is stated clearly
- timeline is stated clearly
- scope is narrow
- client understands this is async

## Before kickoff
- payment is confirmed
- kickoff message is sent
- required access list is requested
- exact bottleneck is confirmed in writing

## Before delivery starts
- the project is marked `won`
- the price is logged
- the source page is logged
- the expected proof opportunity is noted

## Main blocker still outside the repo
- real payment link setup remains the only major external dependency before the first true close
