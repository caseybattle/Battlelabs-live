# Lillian So Pilot Build

## Goal
Create a softer async step before the full application so warm buyers can enter the pipeline without committing immediately to the highest-friction action.

## Problem Summary
From the teardown, the main gap appears to be:
- the full application is doing too much of the qualification work before a lighter async step exists

This matters because:
- some interested buyers are likely dropping before they ever qualify
- follow-up has less opportunity to recover warm interest
- the current flow may be too committed for buyers who are close but not ready

## Pilot Scope
This pilot build focuses on one specific outcome:
- add a lighter pre-application path and follow-up layer

Typical components:
- softer CTA before the full application
- short async fit form
- follow-up sequence for warm but not-ready leads
- routing into the existing application path

## Deliverables
- lighter pre-application entry path
- short qualification form
- follow-up sequence into the existing offer

## Timeline
- target turnaround: 72 hours to 5 days
- delivery model: async only

## Price
- pilot build fee: `$350`

## What is not included
- full application rewrite
- full website rebuild
- major backend system work
- retainer support unless added separately

## Why this scope
The pilot isolates the main front-end leak first: too much friction before a softer async raise-hand step exists.

## Close
If this scope looks right, the next step is a payment link and a simple async kickoff.
