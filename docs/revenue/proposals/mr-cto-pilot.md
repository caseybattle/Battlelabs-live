# Mr. CTO Pilot Build

## Goal
Improve inquiry routing so different buyer types reach the right next step without one broad form doing all the work.

## Problem Summary
From the teardown, the main gap appears to be:
- multiple buyer types are being funneled through a path that is too broad for the different service contexts

This matters because:
- founders, technical leaders, and teams likely need different qualification paths
- broader intake creates more manual sorting after the lead arrives
- the current path may hide higher-fit leads inside a generic inquiry stream

## Pilot Scope
This pilot build focuses on one specific outcome:
- add segmented async qualification and routing for the main buyer categories

Typical components:
- tighter CTA copy tied to buyer type
- segmented form routing
- lighter qualification logic
- follow-up sequence or async review handoff

## Deliverables
- improved inquiry routing across the main buyer categories
- segmented qualification form
- async follow-up logic or review workflow

## Timeline
- target turnaround: 72 hours to 5 days
- delivery model: async only

## Price
- pilot build fee: `$450`

## What is not included
- full website rebuild
- deeper service offer repositioning across the whole site
- custom app or CRM implementation
- retainer support unless added separately

## Why this scope
The pilot is built to reveal whether cleaner routing improves lead quality and reduces manual qualification before any larger build is proposed.

## Close
If this scope looks right, the next step is a payment link and a simple async kickoff.
