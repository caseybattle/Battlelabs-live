# Shay Johnson Pilot Build

## Goal
Sort buyers by readiness before the full application/call path so the front-end flow matches where they actually are.

## Problem Summary
From the teardown, the main gap appears to be:
- the current path leans too heavily on one application/call motion before readiness is sorted

This matters because:
- ready-now and not-yet-ready buyers likely need different next steps
- warmer prospects may leave instead of entering a useful follow-up path
- broader front-end routing reduces follow-up quality

## Pilot Scope
This pilot build focuses on one specific outcome:
- add readiness-based async qualification before the full application/call

Typical components:
- readiness-based CTA cleanup
- short async form
- follow-up sequence tied to buyer readiness
- routing into the stronger-fit path

## Deliverables
- readiness-based front-end inquiry path
- short async qualification form
- follow-up sequence for different readiness levels

## Timeline
- target turnaround: 72 hours to 5 days
- delivery model: async only

## Price
- pilot build fee: `$350`

## What is not included
- full program rewrite
- large backend system implementation
- broad copy overhaul
- ongoing support unless added separately

## Why this scope
The pilot is designed to prove whether better readiness routing improves both capture and follow-up quality without changing the core offer.

## Close
If this scope looks right, the next step is a payment link and a simple async kickoff.
