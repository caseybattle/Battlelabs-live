# Proof Publishing Checklist

## Objective
Turn each finished pilot into reusable proof across the sales system.

## Minimum proof package
- before screenshot
- after screenshot
- original bottleneck summary
- delivered scope summary
- one testimonial or approval line

## Where to use it
- Battle Labs homepage proof section
- async teardown follow-ups
- pilot proposal credibility line
- outbound replies when a prospect asks “have you done this before?”

## Publishing order
1. save the raw proof assets
2. fill out the case-study template
3. extract the shortest usable proof line
4. decide whether it belongs on the homepage yet or only in outreach

## Homepage rule
Only add proof to the homepage if it is:
- real
- understandable quickly
- narrow enough to support the current offer

## Outreach rule
Even if the proof is too small for the homepage, it can still be used in:
- teardown follow-ups
- proposal handoffs
- reply handling

## Pricing rule
After 2 clean wins with usable proof:
- review pilot price
- consider moving the entry point upward if delivery stayed tight
