# Win-State Update Rules

## Purpose
Keep the queue and scoreboard accurate once a lead moves from interest to accepted scope.

## Status transitions
- `qualified` -> `proposal sent`
  when the pilot scope is shared
- `proposal sent` -> `won`
  when the payment link is accepted and payment is confirmed
- `won` -> delivery in progress
  track this in notes until a dedicated fulfillment board exists

## Required notes on a win
Always capture:
- page tag or entry source
- segment
- exact pilot bottleneck
- quoted price
- accepted timeline

## Why this matters
The first win is not just revenue.
It becomes:
- proof
- pricing data
- scope data
- delivery-template data

## First proof capture checklist
After a win, immediately create or capture:
- before screenshot
- bottleneck description
- accepted scope
- price
- eventual after screenshot
