# First Client Fulfillment Checklist

## Objective
Deliver the first paid Battle Labs pilot cleanly enough that it can become proof and a repeatable template.

## Before kickoff
- confirm the single bottleneck being fixed
- confirm the current page or flow being changed
- confirm success means one cleaner next step, not a full rebuild
- collect required links, copy, forms, and access

## Scope guardrails
- one front-end leak
- one pilot outcome
- one follow-up path
- no broad consulting drift

## Build checklist
- review current page and CTA path
- map the new inquiry route
- draft follow-up copy
- connect the form or routing layer
- test the main path end to end
- record async handoff notes

## Delivery checklist
- send a short Loom or written handoff
- explain what changed
- explain what to monitor next
- tell the client what is intentionally not included yet

## Proof checklist
- capture before/after screenshots
- capture the original problem statement
- capture the delivered scope
- request one short testimonial or approval line if the client is happy

## Internal questions after delivery
- did the scope stay narrow enough?
- how long did delivery actually take?
- which steps can become a reusable template?
- should the price move up before the next close?
